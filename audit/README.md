# `audit/` — metrics audit baseline

This directory holds the **point-in-time baseline** produced by the 2026-05-22
metrics audit. The files are auto-generated by `scripts/audit/` and committed
to git so future agents can read the provenance of each per-metric verdict
without re-running the full audit (which is expensive — synthetic corpus
generation, full LPIPS sweep, and a ~5-minute compression sweep).

| File | Generator | Size | Provenance |
|---|---|---|---|
| `manifest.json` | `scripts/audit/build_sample.py` | ~30 KB | Synthetic + real GIF inventory used as the audit corpus. |
| `pilot.csv` | `scripts/audit/pilot.py` | ~90 KB | Per-GIF × per-lossy-level metric table from the pilot sweep. |
| `pilot_decisions.json` | `scripts/audit/pilot.py` | ~6 KB | Per-metric verdicts derived from `pilot.csv`. |
| `sanity.json` | `scripts/audit/sanity.py` | ~562 KB | Identity + pathological + monotonicity check results across the targeted synthetic set. Large because it stores every metric value at every degradation level for every base GIF. |

## What's NOT tracked here

The following are explicitly gitignored (see top-level `.gitignore`):

- `audit/synthetic/` — the generated synthetic-GIF corpus. Regenerate with
  `poetry run python scripts/audit/build_sample.py`.
- `audit/sweep.csv` — the full sweep output (large, derivable). Regenerate
  with `poetry run python scripts/audit/sweep.py` after each fix wave.

## Regenerating the baseline

After any change to `src/giflab/metrics.py`, `src/giflab/enhanced_metrics.py`,
`src/giflab/temporal_artifacts.py`, or `src/giflab/gradient_color_artifacts.py`
that's expected to shift metric values, re-run:

```bash
poetry run python scripts/audit/build_sample.py   # only if synthetic set changed
poetry run python scripts/audit/sanity.py --output audit/sanity.json
poetry run python scripts/audit/sweep.py
poetry run python scripts/audit/pilot.py
poetry run python scripts/audit/report.py
```

Then commit the updated `audit/sanity.json`, `audit/pilot.csv`,
`audit/pilot_decisions.json`, `audit/manifest.json`, and the new dated
report under `docs/metrics-audit/<YYYY-MM-DD>/`.

The full audit report for the current baseline lives at
[`docs/metrics-audit/2026-05-22/report.md`](../docs/metrics-audit/2026-05-22/report.md).
